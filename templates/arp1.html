Comprendre l'ARP Spoofing

Protocole ARP

Le protocole ARP (Address Resolution Protocol) est utilisé pour associer les adresses IP aux adresses physiques (MAC) sur un réseau local. Lorsqu'une machine veut communiquer avec une autre machine sur le même réseau local, elle envoie une requête ARP pour obtenir l'adresse MAC associée à l'adresse IP cible. La machine cible répond avec son adresse MAC. Cette association (IP/MAC) est stockée temporairement dans une table appelée cache ARP.

ARP Spoofing

Dans une attaque ARP Spoofing, l'attaquant (Kali Linux) envoie de fausses réponses ARP aux victimes pour leur faire croire qu'il possède l'adresse MAC de la passerelle ou d'une autre machine cible. Les victimes mettent alors à jour leur cache ARP avec ces fausses informations.

Ainsi, tout le trafic destiné à la passerelle ou à l'autre victime passera d'abord par la machine de l'attaquant. Cette technique permet à l'attaquant d'effectuer une attaque de type Man-in-the-Middle (MITM) en interceptant les échanges réseau.
